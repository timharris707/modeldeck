A menu bar deck for seven accounts

Native SwiftUI menu bar app. Visual language borrowed from the Codex usage popover you like: quiet dark rows, label and reset on the left, thin bar, one confident number on the right. Everything below is rendered at real size — judge it like the actual app.

01Menu bar icon

The glyph is a deck — three stacked bars that double as capacity meters. It's a template icon (auto-adapts to light/dark menu bars), and it carries state without you opening anything. Three treatments:

◐Sat Jul 194:02 PM
A · Glyph only. Quietest. All healthy — plain template icon.
18% ◐Sat Jul 194:02 PM
B · Glyph + worst number. Shows the lowest remaining % across all accounts, only when something drops below 25%. Healthy = number hidden, back to A.
◐Sat Jul 194:02 PM
C · Glyph + status dot. Amber/red dot when any account crosses a threshold. Minimal footprint, no number.

✓ Decided: A by default — plain white template glyph (macOS tints it automatically, like every icon in your menu bar). The number tracks the lowest remaining % across every account and every window — 5-hour or weekly, active or not — and appears in gold only when it drops below the threshold set in Settings; red at critical; hidden again when capacity recovers.

02The popover — two-column deck (✓ decided)

Decided: the two-column deck is the default layout. Claude's accounts on the left, Codex's on the right, each column headed by its brand mark and carrying its own ACTIVE badge — true to how switching really works, since each provider has its own active account. Rows collapse to a single line — name, badge, worst window's bar, % left, next reset. Clicking a row expands its full windows, and that's where Activate lives: it appears only in the expanded state, so collapsed rows stay clean. One click switches that provider's CLI to the account. Bars are blue when healthy, yellow-gold when low, red at critical. A Settings option switches between this two-column layout and the single-column alternate below.

05Adding account #8

The flow that makes growth painless. The app orchestrates it; the actual OAuth login stays in the provider's own browser flow — ModelDeck never sees or stores credentials.

STEP 1
Choose provider & label

Claude or Codex, a friendly name ("Side Project"), a purpose tag, and a color. ModelDeck creates the isolated profile home behind the scenes.

STEP 2
Sign in — provider's own flow

The app kicks off codex login / claude auth login under that profile and opens the browser. You complete Google OAuth exactly as normal.

STEP 3
Verify & land

ModelDeck reads back the authenticated identity, shows "Signed in as dev@…" for confirmation, pulls first usage, and the account appears in the deck.

06Decisions — locked 2026-07-19

QuestionDecision
% left vs % used "% left" everywhere — one convention across both providers.
Popover layout Two-column deck (§02) — Claude column left, Codex column right, expanding account rows. Single-column available as a Settings option; flat list (§03) not selected.
Switch action "Activate" — appears only when an account row is expanded, never on collapsed rows. One click switches that provider's CLI.
Menu bar icon Plain glyph by default; gold % appears when the lowest remaining % across all accounts and windows drops below the configured threshold, red at critical.
Bar color Blue when healthy (like the Claude apps), yellow-gold under threshold, red at critical.
Notifications Yes — macOS banner at a configurable threshold.
"Active" semantics The active account is what the CLI uses for new sessions — Claude Code (terminal or desktop app) picks up the active Claude account; plain codex uses the active Codex account. Already-running sessions are never touched.
CLI versions Settings shows installed vs. latest for Claude Code and Codex CLI, with a one-click Update when outdated. Architecture: Panely's CliToolProbe pattern — one cached probe returns version + availability + auth state (feeding the account health chips too), refreshed in the background, never blocking launch — plus hiverunner's latest-version comparison for the Update button.
Provider marks Official brand logos — Claude starburst in Anthropic clay (#D97757), OpenAI knot in white — now shown in every mockup above.
